  • 指针

    &取地址，%p打印地址，*声明指针

    1. 指针就是内存地址，指针变量是用来存放内存地址的变量
2. 定义指针变量->把变量地址赋值给指针->访问指针变量中可用的地址
3. 声明指针变量，没有确切的地址可以复制，需要赋值NULL，设置为空指针
4. 指针的算术运算，指针数组，指向指针的指针（多级指针），传递指针给函数，从函数返回指针

    #include <bits/stdc++.h>
using namespace std;

int main() {
    system("chcp 65001"); /*控制台乱码*/
    int var = 20;
    int* ip;

    ip = &var;
    printf("var变量的地址: %p\n", &var);

    /*指针变量存储的地址*/
    printf("ip变量存储的地址: %p\n", ip);

    /*使用指针变量访问值*/
    printf("ip变量指向的变量的值: %d\n", *ip);

    return 0;
}

    #include <bits/stdc++.h>
using namespace std;

/*int *a = &a*/
void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main() {
    system("chcp 65001"); /*控制台乱码*/
    
    int a, b;
    a = 1;
    b = 2;

    swap(&a, &b);

    return 0;
}

    指向结构体的指针，访问结构成员用运算符 -> ，普通结构体变量访问成员用 .

    结构体大小的计算遵福以下规则：
1. 第一个成员在与结构休变量偏移量为0的地址处
2. 其他成员变量要对齐到某个数字 （对齐数）的整数倍的地址处。对齐数=min(編译器默认的一个对
齐数，该成员大小)，在VS 环境下默认值为 8，在Linux 环境下默认值为 4
3. 结构体的总大小为最大对齐数（每个成员变量都有一个对齐数）的整数倍
4. 如果嵌套了结构体的情况，被嵌套的结拘体对齐到其自身对齐数的整数倍处（结构体的对齐数就是其内部成员中最大的对齐数)，，此时结构体的整体大小就是所有最大对齐数 （含被嵌套结构体的对齐数）的整数倍
    
// 比如下面结构体的大小为8
struct s1 {
    char ch1; // 0
    char ch2; // 1
    int i; // 对齐到4(整数倍)
    // 最大对齐数是4，需要对齐到8
}

struct s2 {
    char ch1; // 0
    int i; // 4
    char ch2; // 8
    // 最大对齐数是4，需要对齐到12
}

# x86 架构

  • 进制转换

    • 十进制 -> 二进制 / 十六进制： 短除法

    • 二进制 / 十六进制 -> 十进制： 基数乘指数^n

  • 字长

    • 字节 BYTE、字 WORD 两个字节 、双字 DWORD 四个字节 、四字 QWORD 八个字节

  • 端序 数据排列顺序

    • 大端序 数据低位字节放在内存较高地址，符合人的阅读习惯

    • 小端序 数据低位字节放在内存较小地址，符合计算机从低地址向高地址读取数据的方式

    • file 命令

  • x86 - 冯诺依曼体系结构：<u> 运算器、控制器、存储器、输入设备、输出设备 </u>

    或：CPU 控制单元(指令计数器、指令寄存器)、运算单元、寄存器 、System Bus、Memory 进程:指令、数据(污点分析) 、I/O Devices

  • 寄存器

    x86 寄存器总结

    x86 Assembly/X86 Architecture

    8个通用寄存器：32位[数据寄存器(EAX、EBX、ECX、EDX)、变址寄存器(ESI、EDI)、指针寄存器(ESP指向栈顶、EBP栈针基址)]
1个标志寄存器：EFLAGS(CF无符号进位借位、PF、AF、ZF结果为0置1、SF有符号数的符号位、OF有符号进位借位)
6个段寄存器：CS、DS、ES、FS、GS、SS
5个控制寄存器：CR0、CR1、CR2、CR3、CR4
8个调试寄存器：DR0、DR1、DR2、DR3、DR4、DR5、DR6、DR7
4个系统地址寄存器：保护模式寄存器(GDTR、IDTR、LDTR)、TR
其他寄存器：指令指针寄存器EIP、TSC等。

  • 栈 后进先出

    • 存放函数调用信息 (函数返回地址、传递参数、部分寄存器) 和局部变量，可以用栈追踪程序的函数调用过程

    • 程序的栈 是从进程地址空间的 高地址(栈底)->低地址(栈顶)增长 的

  • 堆 动态分配内存

    • 手动 malloc / new 申请任意大小内存，需要在适当的时候用 free / delete 释放内存

    • 从低地址向高地址增长

  • 进程空间布局 从高内存到低内存排列 Linux 进程的内存空间布局

    内核态内存空间、用户态的堆栈、数据段、代码段

  • 常用汇编指令

    • NOP
    • PUSH
    • POP
    • MOV
    • LEA
    • ADD

    修复规则

    • 在Python3.7及以上版本的编译后二进制文件中，头部除了四字节Magic Number，还有四个字节的空位和八个字节的时间戳+大小信息(16bytes)，后者对文件反编译没有影响，全部填充0即可；
    • Python3.3 - Python3.7（包含3.3）版本中，只需要Magic Number和八位时间戳+大小信息(12bytes)
    • Python3.3 以下的版本中，只有Magic Number和四位时间戳(8bytes)，用Winhex修复文件，在头部写入（非覆盖）上述格式的内容，就可以进行反编译了
