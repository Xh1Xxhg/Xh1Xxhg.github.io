# Linux

# 编译命令

  • -m32 指的是生成 32 位程序
  • -fno-stack-protector 指的是不开启堆栈溢出保护，即不生成 canary

# PIE

关闭地址无关可执行文件 PIE（Position Independent Executable），避免在每次加载程序时都变换加载地址，加载基址被打乱，从而不能通过 ROPgadget 等一些工具来帮助解题

gcc -v 查看 gcc 默认的开关情况

如果含有 --enable-default-pie 参数则代表 PIE 默认已开启，需要在编译指令中添加参数 -no-pie

# ASLR

Linux 地址空间分布随机化机制（ASLR）---- 真正打乱基址

echo 0 > /proc/sys/kernel/randomize_va_space 关闭 Linux 系统的 ASLR

  • 0 关闭
  • 1 普通 ASLR 栈基地址、mmap 基地址、.so 加载基地址都将被随机化
  • 2 增强 ASLR 堆基地址随机化

# 栈溢出 Stack Overflow

x86 与 x64 栈的特性存在差异；虚拟地址空间的栈用于保存函数调用信息和局部变量；程序的栈是从进程地址空间的高地址向低地址增长的，一般采用小端序，所以从栈底高地址开始存储，

原理： 缓冲区溢出 程序向栈中某个变量中写入的字节数超过了这个变量本身所申请的字节数，因而导致与其相邻的栈中的变量的值被改变；写数据入栈 + 恶意的数据长度

危害

  • 覆盖程序的返回地址为攻击者所控制的可执行权限的地址

危险函数

  • gets：不检查输入字符串的长度，以回车判断结束 => 栈溢出
  • scanf
  • vscanf
  • sprintf
  • strcpy，字符串复制，遇到 '\x00' 停止
  • strcat，字符串拼接，遇到 '\x00' 停止
  • bcopy

ESP 栈顶指针寄存器 栈指针寄存器 ，EBP 栈底指针寄存器 基址指针寄存器

IDA 获取函数的调用地址 0x0804843B ，按照小端序存储方式为 \x3b\x84\x04\x08 ，触发栈溢出，需要将该字符串输入，同时要利用 pwntools 将 \x3b 作为一个字符输入

# 确定填充长度

计算要操作的地址与要覆盖的地址的距离，打开 IDA，根据其给定的地址计算偏移

变量的索引模式

  • 相对于栈基地址的的索引，可以直接通过查看 EBP 相对偏移获得
  • 相对应栈顶指针的索引，一般需要进行调试，之后还是会转换到第一种类型。
  • 直接地址索引，就相当于直接给定了地址

覆盖需求

  • 覆盖函数返回地址，直接看 EBP 即可
  • 覆盖栈上某个变量的内容，精细计算
  • 覆盖 bss 段某个变量的内容
  • 根据现实执行情况，覆盖特定的变量或地址的内容。

# Off-By-One

特殊的溢出漏洞，程序向缓冲区 堆、栈、bss段、... 中写入时，写入的字节数超过了这个缓冲区本身所申请的字节数并且只越界了一个字节；因为 Linux 的堆管理机制 ptmalloc 验证的松散性，利用简单且危害大。

产生原因

  • 边界验证不严 循环向堆写数据，次数超出导致多写一个Byte
  • 字符串操作

利用

# Chunk Extend and Overlapping

堆漏洞的一种常见利用手法，通过 extend 可以实现 chunk overlapping 的效果

条件

  • 程序中存在基于堆的漏洞
  • 漏洞可以控制 chunk header 中的数据
